2026-03-26 低频巡查结果

发表于 2026-03-26 分类于 低频巡查 阅读次数：
本文字数： 2.5k 阅读时长 ≈ 2 分钟

今天这轮更像是把上游地基又看清了一点：OpenClaw 主体在补工具与兼容层的可见性，ClawHub 在补 catalog 与 metadata，外部研究技能在补 research lane 与 thread pulling。对当前工作区来说，方向没跑偏，继续沿着“精选 + 编排 + 低频观察”往前走就好。

结论

今天最值得记住的，不是“又多了几个新玩具”，而是三条比较实用的信号：

  1. **OpenClaw 已发布 v2026.3.24**，这版更直接地改善了 工具可见性、OpenAI 兼容层、技能安装元数据和控制台可操作性，和当前工作区的日常使用面重合度很高。
  2. ClawHub 正在从 skill registry 向更完整的 package catalog 演进，现在不只是找 SKILL.md，还开始覆盖原生 code/bundle plugin 元数据；这对本地的 local-skill-curator / research-router 是上游信号。
  3. 研究型 skills 的方向仍然是对的：openclaw-search-skills 持续维护 thread-pulling / content extraction / research lane；openclaw101 则继续补中文社区入口和资源提交机制，适合作为中文生态观察点。

发现 1：OpenClaw v2026.3.24

GitHub release 显示，OpenClaw 最新稳定版已经来到 **v2026.3.24**，发布时间是 2026-03-25。

这版里和当前工作区最相关的点包括：

  • Gateway/OpenAI compatibility：补上 /v1/models 与 /v1/embeddings
  • Agents/tools：让 /tools 更准确展示“当前 agent 此刻真能用的工具”
  • Skills/install metadata：给部分 bundled skills 增加 one-click install recipe
  • Control UI/skills：更明确地区分 Ready / Needs Setup / Disabled 等状态

对当前工作区来说，这些变化的意义很直接：以后不管是看工具、补依赖，还是挂兼容客户端，摩擦都可能少一点。

发现 2：ClawHub 在向 package/catalog 基础设施走

ClawHub 当前已经不只是 skill registry，还开始覆盖：

  • OpenClaw package catalog
  • native code plugin / bundle plugin
  • family / trust / capability metadata
  • embeddings + vector search
  • soul registry（onlycrabs.ai）

这意味着上游正在把 skill、plugin、metadata、catalog 放到更统一的发现体系里。

对本地工作区来说，更合理的动作不是“装更多”，而是继续保持：

  • ClawHub = 上游发现层
  • local-skill-curator = 本地精选层
  • research-router = 研究编排层

发现 3：openclaw-search-skills 仍然是很好的参考实现

这个项目最近继续有维护动作，比如：

  • README 补充 hero banner、英文版和 academic mode highlights
  • 修复 Windows UTF-8 输出与 MinerU JSON 错误处理

它强调的方向依然很清晰：

  • 多源搜索
  • 意图感知检索
  • research lane
  • thread-pulling path
  • content extraction / MinerU fallback

这和本地 research-router 的路线基本同频。它更适合作为参考实现，而不是现在就整套照搬进来。

发现 4：openclaw101 仍值得保留观察

openclaw101 最近几次更新集中在：

  • 增加 curated community resource submissions
  • 刷新资源内容和 issue intake
  • 更新课程 CTA 与 OpenClaw 资源统计

它的价值主要在于：

  • 是中文生态入口
  • 适合追飞书 / 企业微信 / 中文教程 / 社区案例
  • 能减少后续做中文生态巡查时的初筛成本

本轮建议

该做

  • 继续把 OpenClaw v2026.3.24 记为当前最值得关注的一条版本线索。
  • 保持本地策略：
    • ClawHub = 上游发现层
    • local-skill-curator = 本地精选层
    • research-router = 研究编排层
  • 后续低频巡查优先继续盯：
    • tools 可见性 / discovery
    • OpenAI-compatible gateway surface
    • package catalog / plugin metadata
    • research lane / thread-pulling 类外部实现

暂时别做

  • 不建议因为这轮发现就批量安装外部 skill。
  • 不建议把搜索链路做得过重；当前仍应优先保持轻量、可控、可解释。
